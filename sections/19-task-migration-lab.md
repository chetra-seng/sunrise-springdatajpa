# Lab: Migrate Task to Spring Data JPA

## Hands-On Practice

---

# What You're Doing

You have a working Task API. Right now it stores tasks in a `ConcurrentHashMap` in memory.

**Your goal:** Connect it to a real PostgreSQL database — without changing the controller or service.

<v-clicks>

**Three files change:**
- `pom.xml` — add JPA + PostgreSQL dependencies
- `Task.java` — add JPA annotations
- `TaskRepository.java` — replace the class with a JPA interface

**Three files stay the same:**
- `TaskServiceImpl.java` — untouched
- `TaskController.java` — untouched
- `TaskMapper.java` — untouched

</v-clicks>

---

# Step 1 — Add Dependencies to pom.xml

Open `pom.xml` and add inside `<dependencies>`:

```xml
<!-- Spring Data JPA -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<!-- PostgreSQL driver -->
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>

<!-- H2 for tests only -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>test</scope>
</dependency>
```

---

# Step 2 — Configure application.properties

Replace `src/main/resources/application.properties` with:

```properties
server.port=9999

# PostgreSQL
spring.datasource.url=jdbc:postgresql://localhost:5432/taskflow
spring.datasource.username=postgres
spring.datasource.password=secret
spring.datasource.driver-class-name=org.postgresql.Driver

# JPA
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
```

Then create the database (or start Docker):

```bash
psql -U postgres -c "CREATE DATABASE taskflow;"
```

# Step 3 — Update Task.java

Open `Task.java` and add JPA annotations. The fields don't change — only the annotations do.

```java
package com.chetraseng.sunrise_task_flow_api.model;

import jakarta.persistence.*;
import lombok.*;
import org.hibernate.annotations.CreationTimestamp;
import java.time.LocalDateTime;

@Entity
@Table(name = "tasks")
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Task {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    private String description;
    private Boolean completed = false;

    @CreationTimestamp
    private LocalDateTime createdAt;
}
```

---

# What Each Annotation Does

| Annotation | SQL Equivalent |
|------------|---------------|
| `@Entity` | Maps this class to a table |
| `@Table(name = "tasks")` | Names the table `tasks` |
| `@Id` | Marks `id` as the primary key |
| `@GeneratedValue(IDENTITY)` | `BIGSERIAL` / auto-increment |
| `@CreationTimestamp` | Set on INSERT, never updated |

---

# Step 4 — Replace TaskRepository.java

Delete everything in `TaskRepository.java` and replace with:

```java
package com.chetraseng.sunrise_task_flow_api.repository;

import com.chetraseng.sunrise_task_flow_api.model.Task;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface TaskRepository extends JpaRepository<Task, Long> {

    List<Task> findByCompleted(boolean completed);

}
```

<v-click>

That's the whole file. Spring generates the implementation for `findAll()`, `findById()`, `save()`, `deleteById()`, and `findByCompleted()`.

</v-click>

---

# Step 5 — Check TaskServiceImpl.java

Open `TaskServiceImpl.java`. You should **not need to change it**, but verify the `delete` method:

```java
// This pattern works with JpaRepository:
public boolean delete(Long id) {
    if (!taskRepository.existsById(id)) return false;
    taskRepository.deleteById(id);
    return true;
}
```

<v-click>

If your current `delete` calls the old `taskRepository.delete(id)` that returned `Boolean`, you'll need to update it to use `existsById` + `deleteById` instead.

</v-click>

# Step 6 — Run and Verify

Start the app:

```bash
./mvnw spring-boot:run
```

Look for this in the logs — it means the table was created:

```
Hibernate:
    create table if not exists tasks (
        id bigint generated by default as identity primary key,
        completed boolean,
        created_at timestamp(6),
        description varchar(255),
        title varchar(255)
    )
```

# Troubleshooting

| Error | Likely Cause | Fix |
|-------|-------------|-----|
| `Connection refused` | PostgreSQL not running | Start postgres or Docker |
| `database "taskflow" does not exist` | DB not created | `CREATE DATABASE taskflow;` |
| `password authentication failed` | Wrong credentials | Check `application.properties` |
| `Table 'tasks' doesn't exist` | `ddl-auto=none` or validate | Set `ddl-auto=update` |
| `Cannot autowire TaskRepository` | Still a class, not interface | Make it `public interface ... extends JpaRepository` |

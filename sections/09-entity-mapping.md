# @Entity Mapping

## Turning Task.java into a Database Table

---

# What @Entity Does

`@Entity` tells JPA: "This Java class maps to a database table."

Hibernate reads the class and generates:

```sql
CREATE TABLE tasks (
    id          BIGINT       GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title       VARCHAR(255),
    description VARCHAR(255),
    completed   BOOLEAN,
    created_at  TIMESTAMP
);
```

You write Java. Hibernate writes SQL.

# Task.java — Before → After

````md magic-move
```java
// Plain POJO — no database awareness
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Task {
    private Long id;
    private String title;
    private String description;
    private Boolean completed = false;
    private LocalDateTime createdAt = LocalDateTime.now();
}
```

```java
// JPA Entity — Hibernate maps this to a database table
@Entity
@Table(name = "tasks")
@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Task {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    private String description;
    private Boolean completed = false;

    @CreationTimestamp
    private LocalDateTime createdAt;
}
```
````

# Each Annotation Explained

```java
@Entity
// "This class is a JPA entity — map it to a table"

@Table(name = "tasks")
// "The table name is 'tasks' (not 'task' — JPA default would be the class name)"

@Id
// "This field is the primary key"

@GeneratedValue(strategy = GenerationType.IDENTITY)
// "Let the database assign the ID (BIGSERIAL / auto-increment)"

@CreationTimestamp
// "Set this field to the current timestamp when the row is first INSERTed"
// "Never update it after that" — Hibernate manages this automatically
```

---

# @CreationTimestamp vs LocalDateTime.now()

```java
// Before: set in Java at object creation time
private LocalDateTime createdAt = LocalDateTime.now();

// After: set by Hibernate at INSERT time
@CreationTimestamp
private LocalDateTime createdAt;
```

<v-clicks>

- `LocalDateTime.now()` runs when `new Task()` is called in Java
- `@CreationTimestamp` runs when Hibernate executes the INSERT statement
- With JPA, the database timestamp is more reliable (consistent across app servers)
- Remove the field initializer — `@CreationTimestamp` sets it for you

</v-clicks>

# Lombok + JPA: One Caution

`@Data` generates `equals()` and `hashCode()` using all fields. With JPA, this can cause issues.

For JPA entities, use `@EqualsAndHashCode(onlyExplicitlyIncluded = true)` and mark only `id`:

```java
@Entity
@Table(name = "tasks")
@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
@Builder
@EqualsAndHashCode(onlyExplicitlyIncluded = true)
public class Task {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @EqualsAndHashCode.Include
    private Long id;

    // ...
}
```

<v-click>

For this course, `@Data` works fine. In production, prefer explicit equals/hashCode on entities.

</v-click>

# @Column — Fine-Grained Column Control

Use `@Column` when you need more control:

```java
@Column(name = "title", nullable = false, length = 255)
private String title;

@Column(columnDefinition = "TEXT")
private String description;

@Column(name = "completed", nullable = false)
private Boolean completed = false;
```

Without `@Column`, JPA uses:
- Column name = field name (camelCase → snake_case in most dialects)
- `nullable = true`
- Default length for strings (255)

Add this to see the schema Hibernate generates:

```properties
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.hibernate.ddl-auto=update
```

# Full Task.java After Migration

```java
package com.chetraseng.sunrise_task_flow_api.model;

import jakarta.persistence.*;
import lombok.*;
import org.hibernate.annotations.CreationTimestamp;
import java.time.LocalDateTime;

@Entity
@Table(name = "tasks")
@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Task {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    private String description;
    private Boolean completed = false;

    @CreationTimestamp
    private LocalDateTime createdAt;

    @ManyToOne
    @JoinColumn(name = "project_id")
    private Project project;
}
```

`@ManyToOne` + `@JoinColumn` adds a `project_id` FK column — many tasks belong to one project.

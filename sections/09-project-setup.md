# Project Setup

## pom.xml and application.properties

---

# Dependencies to Add

Open `pom.xml` and add two dependencies inside `<dependencies>`:

```xml
<!-- Spring Data JPA — JpaRepository, @Entity, @Transactional -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<!-- PostgreSQL JDBC driver — connects Java to PostgreSQL -->
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>
```

<v-click>

`scope=runtime` means the PostgreSQL driver is only needed at runtime, not compile time.

</v-click>

---

# H2 for Tests

Add H2 as a test-only dependency:

```xml
<!-- H2 in-memory database for tests -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>test</scope>
</dependency>
```

<v-click>

`scope=test` means H2 is excluded from the production build. Tests use H2, production uses PostgreSQL.

</v-click>

# application.properties — Development

`src/main/resources/application.properties`:

```properties
# Server
server.port=9999

# PostgreSQL datasource
spring.datasource.url=jdbc:postgresql://localhost:5432/taskflow
spring.datasource.username=postgres
spring.datasource.password=secret
spring.datasource.driver-class-name=org.postgresql.Driver

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
```

---

# application.properties — What Each Setting Does

| Property | Value | What It Does |
|----------|-------|-------------|
| `datasource.url` | `jdbc:postgresql://...` | Where the DB is |
| `datasource.username` | `postgres` | DB login |
| `datasource.password` | `secret` | DB password |
| `ddl-auto` | `update` | Create/update tables at startup |
| `show-sql` | `true` | Log every SQL statement |
| `format_sql` | `true` | Pretty-print the SQL |

# Test application.properties

`src/test/resources/application.properties`:

```properties
# H2 in-memory database for tests
spring.datasource.url=jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# H2 dialect
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.jpa.hibernate.ddl-auto=create-drop

# Show SQL in test logs (optional)
spring.jpa.show-sql=true
```

`create-drop` → create tables at test start, drop them when done.

# Creating the PostgreSQL Database

Before running the app, create the database:

```bash
# Connect to PostgreSQL
psql -U postgres

# Create the database
CREATE DATABASE taskflow;

# Exit
\q
```

Or with Docker:

```bash
docker run --name taskflow-db \
  -e POSTGRES_DB=taskflow \
  -e POSTGRES_PASSWORD=secret \
  -p 5432:5432 \
  -d postgres
```

---

# ddl-auto Options Recap

| Value | Use Case |
|-------|----------|
| `update` | Development — evolves schema safely |
| `create-drop` | Tests — fresh schema per test run |
| `validate` | Production — fail fast if schema is wrong |
| `none` | Production with Flyway/Liquibase migrations |

<v-click>

Never use `create` or `create-drop` in production — it drops all your data on startup.

</v-click>

# Verify the Setup

After adding dependencies and properties, run the app:

```bash
./mvnw spring-boot:run
```

In the startup logs you should see:

```
Hibernate:
    create table if not exists tasks (
        id bigint generated by default as identity primary key,
        completed boolean,
        created_at timestamp(6),
        description varchar(255),
        title varchar(255)
    )
```

And then test with curl:

```bash
curl http://localhost:9999/api/tasks
# []  — empty list from the real database
```
